## 1. 系统概要

### 1.1 目的

为家庭成员提供共享 Claude 订阅的 VSCode AI 助手插件。用户无需各自购买 AI 订阅，通过家庭内网或 VPN 连接到统一的后端服务，即可在 VSCode 内使用 AI 辅助功能。

### 1.2 系统构成

```
┌─────────────────────┐        VPN (OpenVPN)        ┌──────────────────────────┐
│   Mac Pro（户外）    │  ──────────────────────────→ │   Mac Studio（家）        │
│                     │        10.8.0.6:3000         │                          │
│  VSCode 插件         │                              │  Node.js 后端 (PM2)      │
│  · 聊天 UI           │                              │  · /chat 端点            │
│  · SSE 接收          │                              │  · Claude CLI 调用        │
│  · 模型切换          │                              │  · Ollama 调用            │
└─────────────────────┘                              └──────────────┬───────────┘
                                                                    │
                                              ┌─────────────────────┼──────────────┐
                                              ↓                     ↓
                                     ┌──────────────┐    ┌──────────────────┐
                                     │  Claude CLI  │    │  Ollama（本地）   │
                                     │  (claude.ai  │    │  gemma4-E4B 等   │
                                     │   订阅认证)   │    │                  │
                                     └──────────────┘    └──────────────────┘
```

## 2. 构成要素

### 2.1 VSCode 插件（extension/）

- **职责**：用户界面，收发消息，SSE 流式显示
- **运行环境**：各用户的 Mac（VSCode Webview 内）
- **主要功能**：
  - 聊天界面（气泡式 UI）
  - 流式逐字输出（打字机效果）
  - 停止输出按钮
  - 模型切换（Claude / Ollama）
  - 输入法支持（中文/日文 IME）

### 2.2 后端服务（backend/）

- **职责**：接收插件请求，代理调用 LLM，返回 SSE 流
- **运行环境**：Mac Studio，由 PM2 常驻管理
- **主要功能**：
  - `/chat` 端点：接收消息，选择 LLM，SSE 流式返回
  - `/health` 端点：服务健康检查
  - Claude 适配器：通过 Claude CLI 调用
  - Ollama 适配器：通过本地 Ollama API 调用
  - 系统提示词注入：安全限制 + 开发者信息

### 2.3 LLM

| 模型 | 接入方式 | 认证 |
|---|---|---|
| Claude | Claude CLI（`execFile`） | claude.ai 订阅（OAuth） |
| Gemma（Ollama） | HTTP API（localhost:11434） | 不需要 |

## 3. 技术栈

| 层 | 技术 |
|---|---|
| 插件前端 | TypeScript、VSCode Extension API、HTML/CSS/JS（Webview） |
| 后端 | TypeScript、Node.js、Express |
| 通信（插件↔后端） | HTTP + SSE（Server-Sent Events） |
| 通信（户外） | OpenVPN（ym.FamilyDS.com:7853） |
| LLM（Claude） | @anthropic-ai/claude-code CLI |
| LLM（Ollama） | Ollama REST API（OpenAI 兼容） |
| 进程管理 | PM2 |

---

## 4. 网络构成

| 场景 | 连接方式 | 后端地址 |
|---|---|---|
| 家庭局域网内 | 直接连接 | http://192.168.50.x:3000 |
| 户外（VPN） | OpenVPN → Mac Studio | http://10.8.0.6:3000 |

VSCode 插件的后端地址通过设置项 `aiAssistant.backendUrl` 配置。

---

## 5. 安全设计方针

- 后端不对外公开（仅限 VPN 或局域网访问）
- 系统提示词限制 AI 的行为：
  - 禁止透露宿主机的任何信息（路径、用户名、硬件等）
  - 禁止执行文件增删改查操作
  - 仅在被直接询问时告知开发者姓名
