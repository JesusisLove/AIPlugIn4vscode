## 1. 插件侧（extension/）

### 1.1 文件构成

```
extension/src/
├── extension.ts     # 插件入口，注册命令
└── ChatPanel.ts     # 聊天面板（UI + 通信）
```

### 1.2 extension.ts

插件激活时注册命令 `aiChat.open`，绑定快捷键 `Cmd+Shift+A`（Mac）。

```
activate()
  └── registerCommand('aiChat.open')
        └── ChatPanel.createOrShow()
```

### 1.3 ChatPanel.ts

#### 类属性

| 属性 | 类型 | 说明 |
|---|---|---|
| `_panel` | `vscode.WebviewPanel` | WebView 面板实例 |
| `_disposables` | `vscode.Disposable[]` | 资源释放列表 |
| `_currentRequest` | `http.ClientRequest \| null` | 当前 HTTP 请求（停止功能用） |
| `_stopping` | `boolean` | 用户主动停止标志 |

#### 消息流程

```
[Webview JS]                    [插件宿主]                    [后端]
     │                               │                          │
     │── sendMessage ───────────────→│                          │
     │                     _handleUserMessage()                 │
     │                               │── POST /chat ──────────→│
     │← streamStart ─────────────────│← HTTP 响应头 ────────────│
     │← streamToken（循环）───────────│← SSE data ──────────────│
     │← streamEnd ───────────────────│← data: [DONE] ───────────│
     │                               │                          │
     │── stopGeneration ────────────→│                          │
     │                       req.destroy()                      │
     │← streamEnd ───────────────────│                          │
```

#### 输入法支持（中文/日文）

VSCode Webview（Chromium）中，`compositionend` 先于 `keydown` 触发。  
通过 `setTimeout(50ms)` 延迟重置 `isComposing` 标志，实现：  
- 第一次回车：拦截输入法确认（不发送）  
- 第二次回车：正常发送消息

```
compositionstart  → isComposing = true
compositionend    → setTimeout(50ms) → isComposing = false
keydown(Enter)    → isComposing 为 true 时：拦截（输入法确认回车）
                  → isComposing 为 false 时：发送消息
```

#### 停止按钮状态管理

| 状态 | 发送按钮 | 停止按钮 |
|---|---|---|
| 待机中 | 显示 | 隐藏 |
| 发送后～响应中 | 隐藏 | 显示（红色） |
| 响应完成 / 停止后 | 显示 | 隐藏 |

## 2. 后端侧（backend/）

### 2.1 文件构成

```
backend/src/
├── server.ts              # Express 服务器，端点定义
└── adapters/
    ├── claude.ts          # Claude CLI 适配器
    └── ollama.ts          # Ollama API 适配器
```

### 2.2 server.ts

#### 端点一览

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/health` | 健康检查 |
| POST | `/chat` | 聊天（SSE 流式响应） |

#### /chat 请求规格

**请求 Body（JSON）**

| 字段 | 类型 | 必须 | 说明 |
|---|---|---|---|
| `message` | string | ✓ | 用户消息 |
| `model` | string | | `"claude"`（默认）或 `"ollama"` |

**响应（SSE）**

```
Content-Type: text/event-stream

data: {"token":"你好"}
data: {"token":"，有什么"}
data: [DONE]
```

错误时：

```
data: {"error":"错误信息"}
data: [DONE]
```

#### 模型分发逻辑

```typescript
switch (model) {
    case 'ollama': await streamOllama(message, res); break;
    case 'claude':
    default:       await streamClaude(message, res); break;
}
```

### 2.3 claude.ts（Claude 适配器）

#### 处理流程

```
streamClaude(message, res)
  └── callCli(message)
        └── execFile(CLAUDE_PATH,
              ['--print', '--output-format', 'text',
               '--system-prompt', SYSTEM_PROMPT, message],
              { cwd: '/tmp' })          // 工作目录设为 /tmp，防止泄露项目路径
  └── 获取完整响应后，按词单位 SSE 流式发送（每词 12ms 间隔）
  └── 检测客户端断开（res.writableEnded）时立即停止发送
```

#### 环境变量

| 变量 | 说明 |
|---|---|
| `CLAUDE_PATH` | Claude CLI 路径（例：`/Users/kazuyoshi/.local/bin/claude`） |

#### 系统提示词方针

- 不透露宿主机任何信息（路径、用户名、硬件等）
- 拒绝执行文件增删改查操作
- 开发者名（劉義民）仅在被直接询问时回答
- 自我介绍时不主动提及开发者名

### 2.4 ollama.ts（Ollama 适配器）

#### 处理流程

```
streamOllama(message, res)
  └── POST http://OLLAMA_BASE_URL/chat/completions
        请求体: { model, messages: [system, user], stream: true }
  └── 实时接收 SSE 数据块并转发
        json.choices[0].delta.content → data: {"token": "..."}
  └── data: [DONE] 结束
```

#### 环境变量

| 变量 | 说明 | 示例值 |
|---|---|---|
| `OLLAMA_BASE_URL` | Ollama API 基础地址 | `http://localhost:11434/v1` |
| `OLLAMA_MODEL` | 使用的模型名称 | `gemma4-E4B` |

## 6. 错误处理

| 发生位置 | 错误内容 | 处理方式 |
|---|---|---|
| 插件 → 后端 连接失败 | ECONNREFUSED | 在聊天界面显示「⚠️ 连接后端失败」 |
| Claude CLI 执行失败 | 标准错误输出 / 退出码 | 通过 SSE 返回 `{"error": "..."}` |
| Ollama 连接失败 | ECONNREFUSED | 通过 SSE 返回 `{"error": "..."}` |
| 客户端断开（停止按钮） | EPIPE | 通过 `res.writableEnded` 检测后静默退出 |
| 停止时 req.destroy() | Socket 错误 | 通过 `_stopping` 标志抑制，发送 streamEnd |
