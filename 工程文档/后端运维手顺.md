# AI 助手後端 运维手顺

対象机器：**Mac Studio**（10.8.0.6）  
後端路径：`~/Documents/AIPlugIn4vscode/backend`

---

## 首次配置（只需做一次）

### 第一步：编译

```bash
cd ~/Documents/AIPlugIn4vscode/backend
npm run build
```

### 第二步：清除 macOS 安全属性（PM2 运行必须）

```bash
xattr -rc ~/Documents/AIPlugIn4vscode/backend/dist/
```

### 第三步：用 PM2 启动服务

```bash
cd ~/Documents/AIPlugIn4vscode/backend
pm2 start dist/server.js --name "ai-backend"
```

确认运行正常：

```bash
curl http://localhost:3000/health
# 返回 {"status":"ok","version":"0.0.1"} 即为成功
```

### 第四步：保存配置 & 设置开机自启

```bash
pm2 save
sudo env PATH=$PATH:/opt/homebrew/Cellar/node/24.6.0/bin /opt/homebrew/lib/node_modules/pm2/bin/pm2 startup launchd -u kazuyoshi --hp /Users/kazuyoshi
```

> 执行 startup 命令时会要求输入 sudo 密码。完成后 Mac Studio 开机即自动启动後端，无需手动操作。

## 日常操作

| 操作 | 命令 |
|---|---|
| 查看状态 | `pm2 status` |
| 查看日志 | `pm2 logs ai-backend` |
| 停止 | `pm2 stop ai-backend` |
| 重启 | `pm2 restart ai-backend` |
| 手动启动（已停止时）| `pm2 start ai-backend` |

---

## 代码更新后的重新部署

```bash
cd ~/Documents/AIPlugIn4vscode/backend
npm run build
xattr -rc dist/
pm2 restart ai-backend
```

> 每次修改 `backend/src/` 下的代码后，都需要重新 build 再 restart。

---

## 确认服务是否在运行

浏览器访问以下地址（局域网内或 VPN 连接时均可）：

```
http://10.8.0.6:3000/health
```

返回以下内容即为正常运行：

```json
{"status":"ok","version":"0.0.1"}
```

> 直接访问 `http://10.8.0.6:3000` 会出现 404 错误，这是正常的（根路径未定义）。

---

## 确认户外连接可用

1. 确认 Mac Studio 已开机并连接网络
2. Mac Pro 开启 OpenVPN
3. 浏览器访问 `http://10.8.0.6:3000/health`，确认返回正常
4. 插件 backendUrl 设置为 `http://10.8.0.6:3000`
5. 发送消息确认可以正常回应

## 故障排查

**症状：插件报「连接后端失败」**

```bash
# 确认服务是否在运行
pm2 status

# 确认端口是否监听
lsof -iTCP:3000 -sTCP:LISTEN

# 查看错误日志
pm2 logs ai-backend --lines 30
```

**症状：PM2 启动报 EPERM 错误**

```bash
xattr -rc ~/Documents/AIPlugIn4vscode/backend/dist/
pm2 kill
pm2 start ~/Documents/AIPlugIn4vscode/backend/dist/server.js --name "ai-backend"
pm2 save
```
